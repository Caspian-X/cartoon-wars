# Gamespec

## Overview
> A short pitch describing what the game is, its genre, and its core loop, so the reader immediately understands what they're building.

## Core Concepts / Glossary
> Definitions of key terms and entities used throughout the rest of the document, so later sections can reference them without re-explaining.

## Game States
> The distinct top-level modes the game can be in (e.g. menu, playing, paused, game over), what's valid in each, and how they transition to one another.

## Game Flow
> The step-by-step sequence of what happens from launch to end, including the structure of a turn or frame and how states connect in practice.

## Rules
> The specific, unambiguous logic governing how the game behaves, organized by system (e.g. movement, combat, scoring) and numbered for easy reference.

## Entities / Data Model
> The key objects in the game and their properties/relationships (e.g. Player, Enemy, Item), independent of implementation details.

## UI / Feedback
> What the player sees, hears, or otherwise experiences in response to key events and state changes.

## Open Questions / TODOs
> A running list of unresolved design decisions or ambiguities that still need to be settled.
